About this demo

Both service models use the same service and the same service-consuming code. The "Normal" service mode simply injects the service in the caller's normal context. The "Service in Worker" mode provides a proxy instead. The service itself is loaded into a completely distinct angular application context living on another thread.

Instructions

  1. browse to the desired service model
  2. enter a string to be echoed back by simulated service
  3. click echo
  4. observe impact on animation
